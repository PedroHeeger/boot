### Objective:
O presente bootcamp ministrado pela **Escola da Nuvem (EDN)** teve como objetivo aprender sobre os fundamentos da nuvem AWS e seus principais serviços, entendendo como utilizá-los na prática. Além disso, este bootcamp serviu de preparação para a realização do exame AWS Certified Cloud Practitioner (CLF-C01).

### Structure:
- A estrutura do bootcamp da plataforma **EDN** é dividida em módulos e cada módulo contém atividades (cursos).
- Para melhor organização deste bootcamp, a estruturação das pastas acompanhou a estrutura do bootcamp. Dessa forma, foram criadas sub-pastas para cada módulo desse bootcamp, sendo que nas sub-pastas dos módulos estão contidas as pastas ou arquivos dos cursos realizados.
- Nos arquivos de README de cada módulo está descrito o que foi realizado em cada um, e podem ser acessado nos links clicáveis na opção **Bootcamp Strucutre**. Os links que não forem clicáveis, são de cursos que, na sua maior parte ou inteiramente, foram assuntos teóricos e não possuíram materiais.
- A sub-pasta **0-aux** foi criada apenas para armazenar imagens auxiliares para a construção dos arquivos de README.md deste bootcamp. A imagem 01 exibe como ficou a estruturação dos arquivos.

### Development:
O programa **AWS re/Start** foi realizado durante o período de 12 semanas com aulas online e ao vivo. Este era dividido em aulas técnicas e aulas de soft skills. O material técnico para acompanhamento do bootcamp foi disponibilizado em uma plataforma online, contendo um ambiente de laboratório que possibilitava o uso da nuvem da **AWS**. O conteúdo deste material era dividido em módulos conforme apresentado no tópico **Bootcamp Strucutre** acima. Cada módulo continha os seguintes tipos de atividades: conteúdo, questionário, laboratório e demonstração, sendo este último fornecido pela **AWS**. 

Para facilitar minha organização, cada módulo do bootcamp foi armazenado em uma sub-pasta contendo um arquivo README explicando todas as atividades do respectivo módulo. Considerei cada atividade como se fosse um curso. Em algumas dessas atividades foi necessário criar sub-pastas dentro das pastas dos módulos para armazenar os arquivos de código utilizados. As atividades de laboratório era desenvolvido no próprio sandbox que tinha na plataforma do bootcamp. Já as atividades de demonstração foram desenvolvidas na minha própria conta da **AWS** através de arquivos de script **PowerShell** com comandos do **AWS CLI** executado na minha maquina física **Windows**. Abaixo, essas atividades estão listadas com links clicáveis que direciona direto para cada uma no seu respectivo módulo, e com uma pequena descrição do que foi realizado em cada uma.
